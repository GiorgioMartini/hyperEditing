## Pipeline defaults

| Setting | Default | Override |
|---------|---------|----------|
| MG layout | `upper-card` (speaker visible) | `motionGraphic.layout` in beat plan |
| Accent color | `#ff3333` | `project.json` → `motion.accentColor` |
| Font | `Montserrat, sans-serif` | `project.json` → `motion.fontFamily` |
| Beat sync | `anchorPhrase` → transcript word timestamps | Automatic in stage 4 |
| Caption gap | `0.15s` between lines | `project.json` → `captions.interSegmentGapSec` |

## Verification checklist (every video)

After `npm run pipeline`:

1. `cd video-projects/<slug> && npx hyperframes lint`
2. Preview: `npx hyperframes preview` — scrub `?comp=captions` and each `?comp=mg-beat-N`
3. Confirm:
   - No overlapping caption lines during fast speech
   - MG appears when anchor phrase is spoken (±0.2s)
   - Speaker face visible during MG beats (upper card, no full-screen scrim)
   - MG shows grid/chrome styling (not plain sans-serif)
4. Draft render + frame extraction at anchor timestamps (see short-form-video skill)
5. Re-run stages after code changes:
   - `--stage 4` — beat planning / anchor sync
   - `--stage 5` — MG template changes
   - `--stage 6` — compose / caption changes
